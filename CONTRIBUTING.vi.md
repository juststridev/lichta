# Đóng góp cho LichTa

Cảm ơn bạn đã quan tâm và muốn đóng góp cho **LichTa**!

Xin lưu ý rằng **mã nguồn cốt lõi (core code) của thư viện này hiện đang là private**. Repository công khai này chủ yếu đóng vai trò là trung tâm lưu trữ tài liệu (documentation), theo dõi lỗi (issue tracking) và thảo luận của cộng đồng.

## Bạn có thể đóng góp như thế nào?

### 1. Báo cáo lỗi (Report Bugs)
Nếu bạn phát hiện lỗi trong tính toán, format, hoặc components, vui lòng tạo Issue bằng cách sử dụng mẫu **Bug Report**.
Khi báo cáo lỗi, vui lòng cung cấp:
- Mô tả rõ ràng về lỗi.
- Các bước để tái hiện lỗi (tốt nhất là kèm theo link CodeSandbox hoặc StackBlitz).
- Kết quả mong muốn so với thực tế.
- Phiên bản `lichta` và framework bạn đang sử dụng.

### 2. Đề xuất tính năng (Feature Requests)
Bạn có ý tưởng cho một tính năng mới? Chúng tôi rất muốn nghe! Hãy tạo Issue bằng cách sử dụng mẫu **Feature Request**. Vui lòng mô tả chi tiết nhất có thể về use case và lý do tại sao tính năng đó lại hữu ích.

### 3. Cải thiện tài liệu (Documentation)
Bạn hoàn toàn có thể đóng góp trực tiếp vào việc cải thiện tài liệu của chúng tôi!
- Nếu bạn phát hiện lỗi chính tả, nội dung chưa rõ ràng, hoặc muốn thêm ví dụ, đừng ngần ngại tạo Pull Request (PR) chỉnh sửa các file Markdown trong repo này.
- Chúng tôi hỗ trợ tài liệu bằng cả tiếng Anh (`.md`) và tiếng Việt (`.vi.md`). Vui lòng cố gắng cập nhật cả hai nếu có thể, hoặc ghi chú trong PR nếu bạn cần hỗ trợ dịch thuật.

## Quy trình tạo Pull Request cho tài liệu
1. Fork repo và tạo branch của bạn từ `main`.
2. Cập nhật các file markdown liên quan.
3. Đảm bảo tuân thủ các quy tắc format và gắn link (cross-link) hiện có.
4. Gửi Pull Request kèm theo mô tả rõ ràng về những thay đổi.

Chúng tôi rất trân trọng sự giúp đỡ của bạn để làm cho LichTa ngày càng hoàn thiện hơn!
